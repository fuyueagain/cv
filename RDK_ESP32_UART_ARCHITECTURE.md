# RDK X5 与 ESP32S3R8N8 串口协作架构

## 1. 设计目标

| 项目 | 设计结论 |
| --- | --- |
| 系统主控 | `ESP32S3R8N8` 作为总状态机主控 |
| 视觉从控 | `RDK X5` 仅负责视觉识别与结果回传 |
| 通信方式 | `UART /dev/ttyS1`，`115200 8N1`，纯 ASCII 文本，一行一条消息 |
| 资源约束 | RDK X5 只有一个摄像头，因此同一时刻只运行一个视觉模块 |
| 切换策略 | `ESP 发模式命令 -> RDK 启动对应视觉任务 -> 识别完成后立即释放摄像头` |
| 目标 | 协议简单、状态清晰、易调试、便于后续扩展 |

## 2. 硬件连接

根据 RDK X5 官方 UART 文档，40Pin 扩展口默认用户串口为 `UART1`，设备节点为 `/dev/ttyS1`，使用 `3.3V TTL` 电平。

| RDK X5 | ESP32S3R8N8 | 说明 |
| --- | --- | --- |
| `TXD` | `RX` | RDK 发给 ESP |
| `RXD` | `TX` | ESP 发给 RDK |
| `GND` | `GND` | 必须共地 |
| `3.3V TTL` | `3.3V TTL` | 禁止直连 5V 串口 |

## 3. 总体职责划分

| 模块 | 职责 |
| --- | --- |
| ESP32S3R8N8 | 管理待机、菜单、确认、出味、电机/雾化器、UI 刷新、超时与复位 |
| RDK X5 `face_detector.py` | 检测用户是否靠近到预设距离以内 |
| RDK X5 `gesture.py` | 检测 `LEFT / RIGHT / OK` 手势 |
| RDK X5 `rdk_vision_controller.py` | 接收 ESP 命令、调度视觉模块、统一串口发送结果 |

## 4. 推荐状态映射

| 业务阶段 | ESP 动作 | 发给 RDK 的命令 | RDK 视觉行为 | RDK 返回事件 |
| --- | --- | --- | --- | --- |
| 待机 `Standby` | 显示猫咪待机画面 | `FACE_WAIT` | 持续做人脸接近检测 | `START` |
| 菜单浏览 `Browsing` | 切换菜品页 | `INTERACT` | 交替做人脸在场检查 + 手势检测 | `LEFT` / `RIGHT` / `OK` / `LEAVE` |
| 确认选择 `Confirmation` | 高亮并锁定选中项 | `PRESENCE` | 仅监测人是否还在 | `LEAVE` |
| 执行出味 `Execution` | 启动雾化器并倒计时 | `PRESENCE` | 仅监测人是否还在 | `LEAVE` |
| 任意阶段复位 | 释放视觉任务 | `STOP` 或 `RESET` | 停止当前检测并释放摄像头 | `STOPPED` |

## 5. 串口协议

### 5.1 ESP -> RDK 命令

| 命令 | 含义 | 何时发送 |
| --- | --- | --- |
| `PING` | 连通性测试 | 上电自检、联调 |
| `FACE_WAIT` | 进入待机人脸唤醒模式 | 进入待机页后 |
| `INTERACT` | 进入菜单交互模式 | 检测到 `START` 后 |
| `PRESENCE` | 进入在场监测模式 | 进入确认页或执行页后 |
| `STOP` | 停止当前视觉任务 | 任意需要抢占切换时 |
| `RESET` | 同 `STOP`，语义上表示整机复位 | 异常恢复时 |

### 5.2 RDK -> ESP 事件

| 事件 | 含义 | ESP 侧处理建议 |
| --- | --- | --- |
| `READY` | RDK 控制器已启动并监听串口 | ESP 收到后再开始发模式命令 |
| `PONG` | `PING` 应答 | 仅作链路确认 |
| `ACK:FACE_WAIT` | 已切换到待机检测模式 | 可忽略或做日志 |
| `ACK:INTERACT` | 已切换到交互模式 | 可忽略或做日志 |
| `ACK:PRESENCE` | 已切换到在场监测模式 | 可忽略或做日志 |
| `START` | 人脸已到达阈值距离内 | 从待机切到浏览页 |
| `LEFT` | 左切菜单 | 菜单索引减一 |
| `RIGHT` | 右切菜单 | 菜单索引加一 |
| `OK` | 确认当前菜品 | 进入确认页 |
| `LEAVE` | 用户离开 | 直接退回待机页 |
| `STOPPED` | 当前视觉任务已停 | 仅作日志 |
| `ERR:...` | 控制器或识别模块异常 | 记录日志并退回安全状态 |

## 6. RDK 内部调度策略

| RDK 模式 | 调度逻辑 | 摄像头使用策略 |
| --- | --- | --- |
| `FACE_WAIT` | 只调用 `face_detector.detect_face()` | 打开人脸模块，触发后释放 |
| `INTERACT` | 先短时做人脸在场检查，再短时做手势检测，循环执行 | 每次检查完成后立即释放，再切到另一个模块 |
| `PRESENCE` | 周期性做人脸在场检查，连续多次未检测到则认为离开 | 每轮检查后释放 |

## 7. 关键交互细节

| 细节 | 方案 |
| --- | --- |
| 单摄像头冲突 | 所有视觉任务统一由 `rdk_vision_controller.py` 调度，不允许 `face_detector` 与 `gesture` 并行运行 |
| 菜单左右滑动防抖 | `LEFT / RIGHT` 发送后，RDK 等待手势释放，避免用户一直保持手势导致连续翻页 |
| `OK` 确认手势 | `gesture.py` 中要求 `OK` 持续稳定 `0.8s` 才触发 |
| 离场判定 | `PRESENCE` 与 `INTERACT` 模式下，人脸连续多次短超时未出现，则上报 `LEAVE` |
| 串口调试 | 建议所有消息使用大写英文单词，便于串口助手与日志直接观察 |

## 8. ESP32 侧推荐主流程

| 步骤 | ESP 状态 | 发送命令 | 等待事件 | 收到后动作 |
| --- | --- | --- | --- | --- |
| 1 | 上电初始化 | `PING` | `PONG` / `READY` | 串口正常后进入待机 |
| 2 | 待机页 | `FACE_WAIT` | `START` | 切换到菜单浏览并发送 `INTERACT` |
| 3 | 浏览页 | `INTERACT` | `LEFT` / `RIGHT` / `OK` / `LEAVE` | 左右切菜 / 进入确认 / 回待机 |
| 4 | 确认页 | `PRESENCE` | `LEAVE` 或确认倒计时结束 | 离开则回待机，否则进入执行 |
| 5 | 执行页 | `PRESENCE` | `LEAVE` 或 10 秒结束 | 离开回待机，正常结束回浏览页并再次发送 `INTERACT` |

## 9. 当前仓库实现落点

| 文件 | 作用 |
| --- | --- |
| `face_detector.py` | 支持独立人脸检测，也支持被上层控制器调用且不自行发 UART |
| `gesture.py` | 支持独立手势检测，也支持被上层控制器调用且不自行发 UART |
| `rdk_vision_controller.py` | RDK 串口控制主入口，负责接收命令、调度视觉模块、统一发送事件 |
